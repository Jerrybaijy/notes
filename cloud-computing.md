# Cloud computing

[**云计算**](https://cloud.google.com/learn/what-is-cloud-computing?hl=zh-CN)（Cloud computing）就是指通过互联网以服务的形式按需提供计算资源（例如存储和基础设施）。这样，个人和企业就无需自行管理物理资源，而且只需为实际使用的资源付费。

## 云计算部署模型的类型

- [公有云](https://cloud.google.com/learn/what-is-public-cloud)：由第三方云服务提供商运营。
- [私有云](https://cloud.google.com/discover/what-is-a-private-cloud)：由单个组织构建、管理和拥有，并以非公开方式托管在自己的数据中心（通常称为“本地”）内。
- [混合云](https://cloud.google.com/learn/what-is-hybrid-cloud)：结合了公有云和私有云模型，使企业能够利用公有云服务，并仍可保持私有云架构中常见的安全和合规功能。

## 云计算主要服务类型

- [**基础设施即服务**](https://cloud.google.com/learn/what-is-iaas) (IaaS) 可让您按需访问 IT 基础设施服务，包括计算、存储、网络和虚拟化。它提供对 IT 资源的最高级别的控制，与传统的本地 IT 资源最相似。
- [**平台即服务**](https://cloud.google.com/learn/what-is-paas) (PaaS) 提供云应用开发需要的所有硬件和软件资源。采用 PaaS 时，企业无需管理和维护底层基础设施，可以专注于应用开发。
- **软件即服务** (SaaS) 以服务的形式提供完整的应用栈，从底层基础设施到维护，到应用软件本身的更新。SaaS 解决方案通常是最终用户应用，服务和基础设施由云服务提供商管理和维护。

## 三大资源

### 计算 (Compute)

**计算是云的核心“大脑”，负责处理数据和运行指令。**

- **本质：** 云供应商将大量的物理服务器通过虚拟化技术，切分成一个个可以按需分配的单元。
- **常见形式：**
  - **虚拟机 (VM)：** 模拟完整的计算机，如阿里云的 ECS、AWS 的 EC2。
  - **容器 (Containers)：** 更轻量级的运行环境，如 Kubernetes (K8s) 管理的服务。
  - **无服务器计算 (Serverless)：** 用户只写代码，完全不管服务器，如 AWS Lambda 或函数计算。
- **衡量指标：** CPU 核数、内存大小 (RAM)、GPU 算力。

### 存储 (Storage)

**存储是云的“硬盘”，负责永久性或临时性地保存数据。**

- **本质：** 解决数据在不同场景下的存放、读取和备份需求。
- **常见类型：**
  - **块存储 (Block Storage)：** 类似于电脑硬盘，直接挂载给服务器，读写速度最快（如云盘）。
  - **对象存储 (Object Storage)：** 适合存放海量非结构化数据，如图片、视频、备份文件。通过 URL 访问，容量近乎无限（如阿里云 OSS、AWS S3）。
  - **文件存储 (File Storage)：** 像局域网里的共享网盘，支持多个服务器同时挂载（如 NAS）。

### 网络 (Network)

**网络是云的“神经系统”，负责连接计算资源、存储资源以及外部用户。**

- **本质：** 在云端构建一个虚拟的局域网，确保数据的安全传输和高效分发。
- **核心组件：**
  - **虚拟私有云 (VPC)：** 在公有云里为你划出的“私人领地”，确保你的资源与其他用户隔离。
  - **负载均衡 (SLB/ELB)：** 当流量太大时，把访问请求平均分配给多台服务器，防止宕机。
  - **内容分发网络 (CDN)：** 把静态资源缓存到离用户最近的节点，加速访问速度。
  - **专线/VPN：** 连接你的本地办公室和云端数据中心。
